# UAV Forensics Simulation  
### Detection and Analysis of GPS Spoofing & RF Jamming Attacks  
**Author:** Hakan Emre Erolan  
National College of Ireland — MSc Cybersecurity  

---

## 1. Project Overview

This repository implements a **fully reproducible digital-forensics and machine learning framework** for analysing GPS spoofing and RF jamming attacks against UAV systems.

The system simulates a 25-minute UAV flight composed of:

- **Normal flight segment**
- **GPS spoofing attack segment**
- **RF jamming attack segment**

The pipeline integrates:

1. Synthetic telemetry generation  
2. Forensic attack detection using domain-specific thresholds  
3. Supervised ML classification (Random Forest, SVM, Logistic Regression)  
4. Correlation of ML predictions with forensic evidence  
5. Full visual event reconstruction through timelines  

This project supports UAV incident response, forensic investigation, and academic research into cyber-physical threats.

---

## 2. Research Question

**How can forensic methods be improved to reliably detect and analyse GPS spoofing and RF jamming attacks on UAV systems?**

---

## 3. Objectives

1. Review existing UAV forensic frameworks and assess their limitations.  
2. Generate a controlled and reproducible synthetic UAV telemetry dataset.  
3. Detect GPS spoofing and RF jamming using transparent forensic thresholds.  
4. Train supervised machine learning models to classify flight states.  
5. Correlate ML predictions with forensic detections to produce a unified timeline.  
6. Provide an end-to-end forensic simulation pipeline suitable for research and investigation.

## 5. Dataset Description

### **5.1 flight_log.csv**

Generated by `generate_flight_log.py`.  
Contains **1500 telemetry records** at 1 Hz sampling (25 minutes).

Variables include:

- timestamp  
- latitude, longitude  
- altitude  
- speed  
- heading  
- signal_strength  
- gps_drift  

Attack segmentation:

- **0–500 rows → Normal**
- **500–1000 rows → GPS Spoofing**
- **1000–1500 rows → RF Jamming**

---

### **5.2 drone_simulation_dataset.csv**

Created by `generate_dataset.py`.  
Used for machine learning classification.

Includes telemetry features, forensic flags, and clean labels.  
Integrity ensured with SHA-256 hash (`dataset_sha256.txt`).

---

## 6. Forensic Detection

### GPS Spoofing Rule

```
gps_drift > 3.0 meters
```

### RF Jamming Rule

```
signal_strength < -75 dBm
```

Detectors produce logs, CSV event exports, and visual plots.
